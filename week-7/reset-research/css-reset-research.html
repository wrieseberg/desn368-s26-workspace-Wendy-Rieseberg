Josh Comeau — A Modern CSS Reset

Main Idea:

Browsers are much more consistent now. Css support has improved a lot and standards are more reliable. Modern resets focus more on accessibility, usability, and responsive design instead of removing everything. Responsive design became important because people now browse on phones, tablets, and different screen sizes.

What stood out:

Modern CSS resets are less aggressive and improve usability and accessibility instead of just removing all default styles.

Critical Questions

Why reset CSS?

Resets reduce inconsistencies and create a more consistent and predictable starting point. They also remove or fix conflicting browser default styles giving more control over design and layout.

Why use Josh’s version over Eric’s?

  • Preserves useful defaults
  • Better accessibility features
  • Modern responsive techniques

How do you actually implement CSS resets?

The best way to implement CSS resets is to place a reset or normalize stylesheet at the top of your CSS before your main styles. Most modern developers use lightweight resets or normalize approaches that improve consistency while preserving useful browser defaults.

What is a boilerplate?

A boilerplate is a starting template that includes common code and structure for a project. It provides a foundation to build upon and can include things like HTML structure, CSS resets, and basic styling. Boilerplates save time and ensure consistency across projects.

When does a boilerplate help, and when does it hurt?

A boilerplate helps when it provides a solid foundation and saves time by including common code. It can hurt if it includes unnecessary code or if it becomes bloated and difficult to maintain. It's important to choose or create a boilerplate that fits the specific needs of your project, otherwise it can lead to confusion and inefficiency.

Why is normalize.css different from reset, and what is the philosophy shift?

Normalize.css is different from a traditional reset because it doesn't remove all default styles. Instead, it aims to make built in browser styling consistent across different browsers while preserving useful defaults. The philosophy shift is from "removing all styles" to "normalizing styles" for better usability and accessibility.
